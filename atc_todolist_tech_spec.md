# 항공교통관제사(ATC) 맞춤형 종합 근무 일지 & To-Do List 앱 기술명세서 (v3.0)

## 1. 개요 (Product Overview)
- **프로덕트명:** ATC Duty & Task Tracker (가칭)
- **목적:** 분초를 다투는 항공기 관제 업무(To-Do) 관리뿐만 아니라, **관제사의 출퇴근, 섹터 배정 시간, 관제량, 특이사항 로그** 등 핵심 근무 기록(Duty Log)까지 하나로 통합 관리할 수 있는 관제사 전용 종합 워크스테이션 앱입니다.

---

## 2. 사용자 시나리오 (User Scenario)
1. **출근 및 섹터 로그인:** 앱에 접속해 '출근' 버튼을 누르고, 배정받은 **섹터(예: East Sector)와 담당 업무(예: Radar/Coordinator)**를 선택하여 로그인합니다.
2. **관제 할 일 관리:** 관제 구역으로 진입하는 항공기(Flight Strip)를 추가하고 처리하며 실시간으로 업무를 관리합니다.
3. **관제량 자동 누적:** 항공기 관제 카드를 '완료' 처리할 때마다 **'관제량(Traffic)'** 카운터가 자동으로 올라갑니다.
4. **특이사항 기록:** 기상 악화, 장비 이상, 항공기 비상 상황 등 특이사항 발생 시 즉각적으로 시간(UTC)과 함께 로그(Remarks)를 남깁니다.
5. **섹터 교대 및 로그아웃:** 휴식 교대 시 '섹터 로그아웃'을 하여 정확한 관제석 착석 시간을 기록합니다.
6. **퇴근 및 일지 마감:** 업무 종료 시 '퇴근'을 누르면, 일일 관제량, 섹터별 근무 시간, 특이사항이 요약된 **일일 근무 일지(Daily Report)**가 완성됩니다.

## 3. 기능 명세 (Functional Requirements)

### 3.1. [근무 및 로그 관리] (신규 추가 기능)
| 기능 ID | 대분류 | 기능명 | 설명 | 중요도 |
|---|---|---|---|---|
| **F-01** | 근무 | 출퇴근 기록 | 출근(Clock-in) 및 퇴근(Clock-out) 시간 타임스탬프 기록 | 필수 |
| **F-02** | 근무 | 섹터/업무 로그인 | 배정된 섹터명과 담당 업무(예: 레이더, 비행자료 등)의 로그인/로그아웃 시간 기록 | 필수 |
| **F-03** | 통계 | 관제량 집계 | 할 일(비행기) 완료 처리 시 자동 누적 + 수동 관제량 카운트(+, -) 기능 | 필수 |
| **F-04** | 일지 | 특이사항(Remarks) 기록 | 현재 시간(UTC)이 자동 입력되는 텍스트 박스로 이벤트, 장비 이슈 등 로그 작성 | 필수 |
| **F-05** | 리포트| 일일 근무 일지 조회 | 퇴근 시 전체 근무 시간, 섹터 이력, 총 관제량, 특이사항을 모아보는 요약 뷰 | 우대 |

### 3.2. [할 일 및 시간 관리] (기존 핵심 기능)
| 기능 ID | 대분류 | 기능명 | 설명 | 중요도 |
|---|---|---|---|---|
| **F-06** | 시간 | 듀얼 타임 표시 | UTC와 Local Time을 초 단위로 실시간 렌더링 | 필수 |
| **F-07** | 할 일 | Flight Strip 관리 | 항공기 정보 입력, 대기➔진행➔완료 원클릭 상태 전환 | 필수 |
| **F-08** | 알림 | 시간 임박 경고 | 목표 시간 임박 시 카드 색상 변경 및 점멸 효과 제공 | 필수 |

## 4. UI/UX 레이아웃 명세 (Screen Layout)

다중 작업(Multi-tasking) 효율을 높이기 위해 화면을 크게 두 패널로 분리합니다.

*   **[좌측 패널] Duty Log & Stats (근무 기록 영역):** 출퇴근/섹터 로그인 컨트롤 박스, 현재 관제량 카운터, 특이사항(Remarks) 입력 창 및 실시간 타임라인(로그) 표출.
*   **[우측 패널] Task Tracker (관제 할 일 영역):** 듀얼 시계 및 실시간 Flight Strip 카드 리스트(임박 시간순 자동 정렬).
*   **디자인 규약:** 완벽한 다크 모드(배경 `#121212`), 고정폭(Monospace) 폰트, 색상 코딩(비상 Red, 주의 Amber, 일반 Green) 유지.

## 5. 데이터 스키마 (Data Schema)

앱의 데이터를 체계적으로 관리하기 위해 스키마를 4개로 확장합니다.

### 5.1. `ShiftSession` (출퇴근 세션)
| 필드명 | 데이터 타입 | 설명 | 예시 |
|---|---|---|---|
| `clockIn` | String | 출근 시간 (UTC) | `09:00:00Z` |
| `clockOut` | String | 퇴근 시간 (UTC) | `18:00:00Z` |
| `totalTraffic`| Number | 일일 총 관제량 누적치 | `142` |

### 5.2. `SectorLog` (섹터 로그인 기록)
| 필드명 | 데이터 타입 | 설명 | 예시 |
|---|---|---|---|
| `sectorName` | String | 관제 구역명 | `East Sector` |
| `role` | String | 담당 업무 | `Radar`, `Coordinator` |
| `loginTime` | String | 착석(로그인) 시간 | `10:00:00Z` |
| `logoutTime` | String | 이석(로그아웃) 시간 | `11:30:00Z` |

### 5.3. `RemarkLog` (특이사항 기록)
| 필드명 | 데이터 타입 | 설명 | 예시 |
|---|---|---|---|
| `timestamp` | String | 특이사항 발생 시간 | `10:45:00Z` |
| `content` | String | 특이사항 내용 | `VHF 송출 불량으로 예비 장비 전환` |

### 5.4. `Task` (할 일 / 비행기록표)
*   `id`, `callsign`, `taskType`, `targetTime`, `status`, `priority` (기존 유지)

## 6. 개발 단계 및 마일스톤 (Milestones)

- **Phase 1 (근무/로그인 시스템):** 화면 좌/우 레이아웃 구성. 좌측에 출퇴근 버튼, 섹터/업무 선택 폼, 로그인 타임스탬프 기록 기능(LocalStorage 연동) 구현.
- **Phase 2 (할 일 및 관제량 연동):** 우측 패널에 Flight Strip (카드) UI 및 듀얼 시계 구현. 카드를 '완료' 처리할 때마다 좌측 패널의 '관제량' 카운터가 +1 증가하도록 연동.
- **Phase 3 (특이사항 및 마감):** 좌측 하단에 특이사항(Remarks) 입력창 및 로그 뷰어 추가. 퇴근 버튼 클릭 시 전체 데이터(출퇴근, 섹터 이력, 관제량, 로그)를 합쳐 보여주는 모달(Report) 구현.

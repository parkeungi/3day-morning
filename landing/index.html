Capabilities

관제 현장이 요구하는 정확한 도구만.

일반 To-Do 앱이 아닙니다. 근무 일지(Duty Log)와 작업 트래커(Task Tracker)를 한 도구로 결합하여, 다음과 같은 핵심 기능을 제공합니다.

SEC · 02 / FEATURES
01 · SHIFT
⏱

출퇴근 클럭

한 번의 클릭으로 근무 시작/종료를 UTC 기준으로 기록. 누적 근무 시간이 자동으로 계산됩니다.

02 · SECTOR
▣

섹터 착석 / 이석

구역과 역할(Radar 등)을 선택해 착석. 섹터별 소요 시간이 자동 누적되어 교대 흐름이 보입니다.

03 · STRIP
✈

Flight Strip 작업대

콜사인 단위 카드로 대기 → 진행 → 완료 워크플로우. 시각적으로 즉시 우선순위를 파악합니다.

04 · TRAFFIC
#

누적 관제량 카운터

스트립 완료 시 자동 +1. 필요 시 수동 ± 보정도 가능합니다. 일일 통계로 그대로 활용.

05 · ALERT
!

시간 위반 경고

목표 시각 5분 전 주황(주의), 1분 전·초과 시 빨강 펄스(위험). 시선이 자연스럽게 끌립니다.

06 · LOG
≡

특이사항 즉시 로그

이벤트 발생 즉시 한 줄 입력. UTC 타임스탬프가 자동 첨부되어 사후 보고에 즉시 사용됩니다.

07 · REPORT
↗

일일 리포트 + 복사

퇴근 시 자동 요약 모달. Copy as Text 한 번으로 메신저·메모장에 즉시 붙여넣기.

08 · UI
☾

다크 / 라이트

시스템 설정 자동 감지 + 수동 토글. 장시간 근무에도 편안한 가독성을 제공합니다.

09 · DATA
⌬

안전한 로컬 저장

모든 데이터는 브라우저 LocalStorage에 저장. 새로고침·재방문 시 그대로 복구됩니다.

Daily Workflow · 5 Steps

출근부터 퇴근까지, 5단계.

실제 관제사의 하루 흐름을 그대로 반영했습니다. 각 단계는 독립적으로도 동작하며, 전체 흐름을 따라가면 자연스럽게 완전한 근무 일지가 생성됩니다.

SEC · 03 / FLOW
  1. 01
    SHIFT-IN

    출근 보고

    좌측 도크의 ▶ 출근하기를 클릭. 상태 바가 ‘근무 중’으로 전환되며 시간이 UTC로 기록됩니다.

  2. 02
    SIGN-ON

    섹터 착석

    구역(예: East Sector)과 역할(Radar)을 선택하고 ▶ 착석. 섹터별 시간 자동 누적.

  3. 03
    STRIP / OPS

    스트립 처리

    콜사인·목표 시각으로 추가 → Start → Complete. 완료 시 누적 관제량 자동 +1.

  4. 04
    REMARK

    특이사항 기록

    이벤트 발생 즉시 한 줄 입력 → 기록. UTC 타임스탬프가 자동 첨부됩니다.

  5. 05
    SHIFT-OUT

    퇴근 + 리포트

    ■ 퇴근하기 → 일일 리포트 모달 자동 오픈 → Copy as Text로 즉시 복사.

Visual Alert Logic

시각적 시간 위반 경고 — 보지 않아도 보입니다.

다양한 항공기를 동시에 관제할 때 가장 먼저 사라지는 것은 ‘시간 감각’입니다. Flight Strip 카드는 목표 시각이 가까워질수록 색상과 펄스로 자연스럽게 시선을 잡아끕니다.

SEC · 04 / ALERTS
NORMAL · > 5 MIN

여유 상태

목표 시각까지 5분 이상 여유가 있는 경우. 평상 색상으로 표시되어 시야의 노이즈가 없습니다.

CAUTION · ≤ 5 MIN

주의 (주황)

목표 시각 5분 전부터 주황색 경고. 다음 처리할 카드를 자연스럽게 인식할 수 있습니다.

DANGER · ≤ 1 MIN / OVER

위험 (빨강 · 펄스)

1분 이내 또는 초과 시 빨간색 점멸. 다른 화면을 보고 있어도 주변 시야로 즉시 인지됩니다.

Q&A · For Answer Engines

자주 묻는 질문

도구 선택 전 확인하실 만한 정보를 정리했습니다. AI 검색 / 음성 비서가 직접 답변에 인용할 수 있도록 구조화 데이터로 작성되었습니다.

SEC · 05 / FAQ
ATC Duty & Task Tracker는 어떤 도구인가요?
항공교통관제사(ATC)를 위한 단일 페이지 워크스페이스입니다. 출퇴근 클럭, 섹터 착석, Flight Strip 처리, 누적 관제량, 특이사항 로그를 한 화면에서 통합 관리하여, 여러 도구를 오가지 않고 근무 일지와 작업 트래커를 동시에 운영할 수 있습니다.
설치가 필요한가요? 어디서 사용하나요?
별도의 설치가 필요하지 않습니다. parkeungi.github.io/3day-morning 접속 즉시 사용 가능한 웹 앱으로, Chrome / Edge / Safari / Firefox 모든 최신 브라우저에서 동작합니다.
제 데이터는 어디에 저장되나요? 안전한가요?
모든 데이터는 사용 중인 브라우저의 LocalStorage(키: atc-tracker:v1)에만 저장됩니다. 외부 서버로 전송되지 않으며, 새로고침·창 닫기 후에도 다음 접속 시 자동 복구됩니다. 다른 기기와는 동기화되지 않습니다.
Flight Strip이 시간을 위반하면 어떻게 표시되나요?
목표 시각 5분 전에는 카드가 주황색(주의)으로 강조되고, 1분 전 또는 초과 시에는 빨간색으로 점멸(펄스)하여 즉각적인 시각 경고를 제공합니다. 다른 화면을 보고 있어도 주변 시야로 인지됩니다.
누적 관제량(Traffic)은 어떻게 집계되나요?
Flight Strip 카드를 Complete 상태로 전환할 때마다 자동으로 +1 됩니다. 필요 시 우측 완료 레인 헤더의 + / − 버튼으로 수동 보정도 가능합니다.
퇴근 후 일일 리포트는 어떻게 활용하나요?
■ 퇴근하기 클릭 시 모달이 자동으로 열리며, 출퇴근 시간·섹터 이력·총 관제량· 특이사항이 깔끔하게 요약됩니다. Copy as Text 버튼으로 클립보드에 복사하여 다른 보고서, 메신저, 메모장에 즉시 붙여넣을 수 있습니다.
다크 모드를 지원하나요?
네. 시스템 설정을 자동 감지하며 상단 바의 ☀ / ☾ 토글로 수동 변경할 수 있습니다. 선택한 테마는 영구 저장되어 다음 접속 시 그대로 유지됩니다.
어떤 기술 스택으로 만들어졌나요?
React 18 + Vite 5 기반으로, JavaScript(JSX)와 CSS Modules를 사용했습니다. 외부 상태 관리 라이브러리 없이 useReducer + useContext만으로 도메인 상태를 운영하며, JetBrains Mono 폰트로 시간/숫자 가독성을 강조했습니다.
모바일에서도 동작하나요?
기본적으로 데스크톱·태블릿(가로) 환경에 최적화되어 있습니다. 모바일에서도 접근 자체는 가능하나, 항공기 스트립을 다수 처리하는 메인 작업대는 큰 화면에서 사용하실 것을 권장합니다.
